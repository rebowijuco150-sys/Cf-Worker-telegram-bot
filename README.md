# Cf-Worker-telegram-bot

این مخزن شامل یک Cloudflare Worker است که به‌عنوان Webhook برای یک ربات تلگرام عمل می‌کند. این Worker به‌سادگی آپدیت‌های تلگرام را دریافت می‌کند، پیام‌ها و دکمه‌های تعاملی را پردازش می‌کند، پاسخ می‌فرستد و در صورت نیاز «آخرین پیام» هر کاربر را در Workers KV ذخیره می‌کند. همچنین لاگ‌ها را در کنسول ثبت می‌کند و در صورت تنظیم می‌تواند لاگ‌ها را به یک چت تلگرامی ارسال کند.

---

## امکانات
- پاسخ به دستورات ساده: `/start` و `/help` و بازگردانی متن‌های دیگر (echo)
- نمایش یک کیبورد داخلی (inline keyboard) در پیام `/start` با دو دکمه:
  - «پینگ» (`callback_data` = `ping`)
  - «آخرین پیام من» (`callback_data` = `last_message`)
- پردازش `callback_query` و پاسخ‌دهی مناسب
- گزینهٔ محافظت با توکن مخفی (هدر `X-Telegram-Bot-Api-Secret-Token`) در صورت فعال‌سازی
- ذخیرهٔ آخرین پیام هر کاربر در Workers KV (binding با نام `SESSIONS`)
- لاگ‌گیری در کنسول Cloudflare و ارسال لاگ به یک چت تلگرامی در صورت تنظیم (مقدار `LOG_CHAT_ID`)
- استفاده از `ctx.waitUntil` برای انجام کارهای پس‌زمینه و بازگشت سریع پاسخ HTTP به تلگرام

---

## پیش‌نیازها
- حساب Cloudflare با دسترسی به Workers
- توکن ربات تلگرام (`BOT_TOKEN`)
- (اختیاری) Workers KV برای نگهداری «آخرین پیام» کاربران
- (اختیاری) یک چت یا گروه تلگرامی برای دریافت لاگ‌ها (`LOG_CHAT_ID`)

## راه‌اندازی در Cloudflare (راهنمای گام‌به‌گام)
1. وارد داشبورد Cloudflare شوید و Worker مورد نظر را باز کنید.
2. به بخش Settings → Variables یا Environment Variables بروید و متغیرهای زیر را اضافه کنید:
   - `BOT_TOKEN` (الزامی): توکن ربات تلگرام (مثال: `123456:ABC...`).
   - `SECRET_TOKEN` (اختیاری، ولی توصیه‌شده): اگر می‌خواهید webhook امن‌تر باشد. هنگام ست کردن webhook باید همین مقدار را به‌عنوان پارامتر `secret_token` ارسال کنید.
   - `LOG_CHAT_ID` (اختیاری): اگر می‌خواهید لاگ‌ها به یک چت تلگرامی فرستاده شوند، مقدار `chat_id` آن را وارد کنید. برای گروه‌ها مقدار معمولاً با `-100` شروع می‌شود.
3. برای فعال کردن قابلیت «آخرین پیام»، یک Workers KV namespace بسازید و آن را به binding با نام `SESSIONS` وصل کنید:
   - در داشبورد Cloudflare به Workers → KV → Create namespace بروید.
   - در قسمت Bindings نام را `SESSIONS` قرار دهید (یا در صورتی که نام دیگری می‌گذارید، کد را مطابق آن تغییر دهید).
4. Worker را Deploy کنید. می‌توانید از داشبورد استفاده کنید یا با ابزار `wrangler` دستور `wrangler publish` را اجرا کنید.

## تنظیم Webhook در تلگرام
پس از دیپلوی، آدرس Worker (برای نمونه `https://myworker.youraccount.workers.dev`) را یادداشت کنید و آن را به تلگرام معرفی کنید.

- اگر `SECRET_TOKEN` ندارید:

```
https://api.telegram.org/bot<BOT_TOKEN>/setWebhook?url=https://YOUR-WORKER.workers.dev
```

- اگر `SECRET_TOKEN` فعال است:

```
https://api.telegram.org/bot<BOT_TOKEN>/setWebhook?url=https://YOUR-WORKER.workers.dev&secret_token=<SECRET_TOKEN>
```

برای بررسی وضعیت webhook از آدرس زیر استفاده کنید:

```
https://api.telegram.org/bot<BOT_TOKEN>/getWebhookInfo
```

نکته: اگر Worker را روی مسیر خاصی میزبانی کرده‌اید (مثلاً `https://.../telegram`) همان مسیر را در مقدار `url` قرار دهید.

## تست سریع
1. در تلگرام به ربات پیام `/start` بفرستید. باید پیامی با متن زیر دریافت کنید:

   سلام 👋 ربات آماده‌ست!

   همراه با دو دکمه: «پینگ» و «آخرین پیام من».
2. روی دکمه «پینگ» بزنید؛ پیام باید به `Pong! 🏓` تغییر کند یا یک پیام جدید برای شما ارسال شود.
3. روی دکمه «آخرین پیام من» بزنید؛ اگر `SESSIONS` فعال باشد و قبلاً پیامی از شما ذخیره شده باشد، آن پیام نمایش داده می‌شود.

---

## لاگ‌گیری
- لاگ‌ها با `console.log` و `console.error` در Cloudflare ثبت می‌شوند و از داشبورد قابل مشاهده‌اند.
- اگر مقدار `LOG_CHAT_ID` تنظیم شده باشد، لاگ‌ها نیز به آن چت تلگرامی ارسال می‌شوند. ارسال لاگ با `ctx.waitUntil` انجام می‌شود تا تأخیر در پاسخ‌دهی ایجاد نشود.

---

## متغیرهای محیطی (جمع‌بندی)
- `BOT_TOKEN` — توکن ربات (الزامی)
- `SECRET_TOKEN` — توکن مخفی برای بررسی هدر درخواست‌ها (اختیاری)
- `SESSIONS` — binding برای Workers KV (اختیاری)
- `LOG_CHAT_ID` — شناسه چت برای ارسال لاگ‌ها (اختیاری)

---

## خطایابی (مشکلات رایج)
- دریافت `401 Unauthorized` از تلگرام → معمولاً `BOT_TOKEN` اشتباه است.
- دریافت `403 Forbidden` از Worker → اگر `SECRET_TOKEN` تنظیم شده و هدر ارسالی با آن مطابقت ندارد.
- ربات پاسخی نمی‌دهد → با `getWebhookInfo` وضعیت webhook را بررسی کنید و از صحیح بودن URL اطمینان یابید.
- ذخیره یا خواندن از KV کار نمی‌کند → مطمئن شوید namespace ساخته و به binding `SESSIONS` متصل شده است.
- دریافت پاسخ‌های `429 Too Many Requests` → تعداد درخواست‌ها را کاهش دهید یا مکانیزم retry/backoff اضافه کنید.

## نکات امنیتی
- `SECRET_TOKEN` را فعال کنید و هنگام setWebhook از پارامتر `secret_token` استفاده کنید.
- `BOT_TOKEN` را در مخزن عمومی قرار ندهید؛ از Secrets یا Environment Variables استفاده کنید.
